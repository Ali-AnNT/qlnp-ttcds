# Phase 3: Update Source Files

## Overview

Add `lma-` prefix to all Tailwind utility classes across 85 source files (556 static + 125 dynamic className + 9 CVA definitions + Calendar/Sonner classNames props). This is the most labor-intensive phase.

## Requirements

- Functional: Every Tailwind utility class must have `lma-` prefix
- Functional: `group`, `peer`, custom classes remain unprefixed
- Non-functional: CVA variant strings, Calendar/Sonner classNames objects must be handled

## Architecture

### Prefix Rules (Tailwind v3 with `prefix: "lma-"`)

Prefix goes AFTER variant modifiers, BEFORE utility name:

| Original | Prefixed | Notes |
|----------|----------|-------|
| `flex` | `lma-flex` | Basic utility |
| `bg-primary` | `lma-bg-primary` | Color utility |
| `hover:bg-primary` | `hover:lma-bg-primary` | Variant + prefix |
| `sm:grid-cols-2` | `sm:lma-grid-cols-2` | Responsive + prefix |
| `dark:bg-muted` | `dark:lma-bg-muted` | Dark mode + prefix |
| `data-[state=open]:bg-accent` | `data-[state=open]:lma-bg-accent` | Data attribute + prefix |
| `-mt-8` | `-lma-mt-8` | Negative: dash before prefix |
| `!font-bold` | `!lma-font-bold` | Important: bang before prefix |
| `group-hover:text-sm` | `group-hover:lma-text-sm` | Group variant + prefix |
| `group/sidebar` | `group/sidebar` | NOT prefixed — group marker |
| `peer` | `peer` | NOT prefixed — peer marker |

### Classes NOT Prefixed

| Class | Reason |
|-------|--------|
| `group`, `group/sidebar`, `group-data-[...]` | Tailwind grouping markers |
| `peer` | Tailwind peer marker |
| `qlnp-app` | Custom scoping class (removed in Phase 2) |
| `toaster`, `toast` | Sonner library classes |
| Custom non-Tailwind classes | Not generated by Tailwind |

### Red Team Findings to Address

1. **CVA variant definitions** (9 total): `buttonVariants`, `badgeVariants`, `sheetVariants`, `toastVariants`, `toggleVariants`, `sidebarMenuButtonVariants`, `navigationMenuTriggerStyle`, `alertVariants`, `labelVariants` — all variant strings need prefix

2. **Calendar classNames prop**: `react-day-picker` receives `classNames` as an object with 15+ key-value pairs of class strings — these bypass `cn()` and must be manually prefixed

3. **Sonner classNames prop**: `toastOptions.classNames` is an object prop — bypasses `cn()`

4. **group-[...] selectors**: `group-[.toaster]` targets a CSS class on a parent element. The `group` marker itself is NOT prefixed, but `group-hover:lma-*` utilities on children ARE prefixed. Verify DOM-class-to-CSS-selector coupling.

5. **[&_...] arbitrary selectors**: 25+ occurrences — these are arbitrary descendant selectors, prefix goes inside: `[&_svg]:lma-size-4`

### Step 2: Manual prefix migration (per-file approach)

> **Validation decision:** Use manual-only migration (no cnat). cnat v0.0.7 is low maturity; manual approach provides more control.

Work through files by category, prefixing all Tailwind utility classes with `lma-`:

**Category order (easiest → hardest):**
1. **App-level** (2 files): `App.tsx`, `NotFound.tsx`
2. **Hooks** (1 file): `use-toast.ts`
3. **UI components** (47 files): `src/shared/ui/*.tsx`
4. **Feature components** (38 files): `src/features/*/components/*.tsx`

**Prefix rules (per phase header table):**
- `flex` → `lma-flex`
- `hover:bg-primary` → `hover:lma-bg-primary`
- `sm:grid-cols-2` → `sm:lma-grid-cols-2`
- `-mt-8` → `-lma-mt-8`
- `!font-bold` → `!lma-font-bold`
- `group/sidebar` → stays `group/sidebar` (NOT prefixed)
- `peer` → stays `peer` (NOT prefixed)
- `[&_svg]:size-4` → `[&_svg]:lma-size-4`
- `data-[state=open]:bg-accent` → `data-[state=open]:lma-bg-accent`

**Strategy per file type:**
- **Static className strings:** Find-replace with prefix rules
- **CVA variant strings:** Same prefix rules applied inside variant objects
- **cn() calls:** Same prefix rules, but inside `cn()` function args
- **Calendar classNames prop:** Object values, each string prefixed individually
- **Sonner classNames prop:** Object values, each string prefixed individually

### Step 5: Handle Calendar classNames prop

In `calendar.tsx`, the `classNames` prop is an object with 15+ string values. Each string contains Tailwind classes that must be individually prefixed:

```tsx
// Before
classNames={{ months: "flex", day: "h-9 w-9 ..." }}

// After
classNames={{ months: "lma-flex", day: "lma-h-9 lma-w-9 ..." }}
```

### Step 6: Handle Sonner classNames prop

In `sonner.tsx`, `toastOptions.classNames` is an object. Prefix all Tailwind classes in its values.

### Step 7: Verify group/peer selectors

Search for `group-[` and `peer-[` patterns. Verify that:
- `group` and `peer` markers remain unprefixed
- `group-hover:lma-*` utilities on children have prefix

### Step 8: Verify [&_...] arbitrary selectors

Search for `[&` patterns. Verify prefix placement:
- `[&_svg]:lma-size-4` (correct — prefix after arbitrary selector)
- NOT `lma-[&_svg]:size-4` (wrong)

### Step 9: Verify shadcn data-variant patterns

Known bug area (shadcn issue #7436). Manually verify these components:

- `sidebar.tsx` — `group-data-[variant=floating]:*` patterns
- `sheet.tsx` — `data-[state=open]:*` patterns
- `dialog.tsx` — `data-[state=open]:*` patterns
- `dropdown-menu.tsx` — `data-[state=open]:*` patterns
