# Brainstorm Report: QLNP Schema Design with Existing System Tables

**Date:** 2026-05-13 | **Branch:** rebuid-bundle | **Updated:** EF Core + Scaffold

## Source Context

Đọc BRD (`docs/vision/brd.md`) + SRS (`docs/vision/srs.md`) + kiểm tra database thực tế (`VI_NGHIPHEP` trên SQL Server 192.168.1.13:1439).

## Key Discovery

SQL Server 2016 instance có **USER_MASTER** (82 users) và **DM_DONVI** (144 đơn vị phân cấp) — hệ thống có sẵn, không thể tạo mới từ đầu.

## Decisions (final)

| # | Mục | Quyết định | Rationale |
|---|-----|-----------|-----------|
| 1 | Project path | `packages/api/` | Monorepo, cạnh `packages/web/` |
| 2 | Project name | `QLNP.Api` | PascalCase .NET convention |
| 3 | Framework | FastEndpoints + VSA | REPR pattern, dễ test, theo code-standards |
| 4 | ORM | **EF Core** | Code First, migration management |
| 5 | PK type | **bigint** | Đồng bộ USER_MASTER (User_MasterID), DM_DONVI (DonViID) |
| 6 | Auth | **API Gateway SSO** | Gateway validate SSO trước khi forward → API chỉ đọc user từ headers |
| 7 | User table | **Không tạo** — dùng USER_MASTER | Scaffold từ DB, đọc qua EF, không duplicate |
| 8 | Dept table | **Không tạo** — dùng DM_DONVI | Scaffold từ DB, filter theo cây phân cấp |
| 9 | Role | **user_roles** riêng | Config từ QTHT, không từ SSO claims |
| 10 | Embed mode | **Vẫn giữ** | iframe + postMessage JWT exchange |
| 11 | .NET version | 9.0 (cài mới) | Hiện tại chỉ có 8.0 |
| 12 | Table prefix | Không prefix | user_roles, leave_types, leave_requests, leave_balances, leave_config |
| 13 | Scaffold | USER_MASTER + DM_DONVI toàn bộ cột | `dotnet ef dbcontext scaffold` |
| 14 | Migration | Exclude bảng cũ, chỉ migrate bảng mới | Bảng cũ = external, không quản lý qua EF migration |

## Database Schema (5 QLNP tables, Code First)

```sql
-- Bảng gán role QLNP (không có trong USER_MASTER)
user_roles (
    user_id BIGINT PRIMARY KEY,  -- FK → USER_MASTER.User_MasterID
    role NVARCHAR(10) NOT NULL CHECK (role IN ('CB.PCM','LD.PCM','GD.PGD','QTHT'))
)

leave_types (id BIGINT IDENTITY PK, name, code UNIQUE, default_days, description, is_active)
leave_balances (id BIGINT IDENTITY PK, user_id FK→USER_MASTER, leave_type_id FK, year, total_days, used_days, UNIQUE(user_id, leave_type_id, year))
leave_requests (id BIGINT IDENTITY PK, user_id FK→USER_MASTER, leave_type_id FK, start_date, end_date, total_days, reason, status CHECK, approved_by FK→USER_MASTER, approved_at, rejected_reason, created_at, updated_at)
leave_config (id BIGINT IDENTITY PK, leave_type_id FK, approval_level CHECK>=1, approver_role)
```

## Key Differences from Original BRD

| Mục | BRD gốc | Thiết kế cuối |
|-----|---------|--------------|
| Số bảng | 6 | 5 |
| employees | Có, UUID PK | **Không** — scaffold USER_MASTER |
| departments | Có, UUID PK | **Không** — scaffold DM_DONVI |
| PK type | UUID | bigint |
| Auth | BCrypt + JWT self-issue | API Gateway SSO |
| ORM | Dapper | **EF Core** + scaffold |
| NuGet | 5 packages | 4 (FastEndpoints, EF Core x3) |
| User/Dept APIs | 8 endpoints | 0 (hệ thống cũ lo) |

## Implementation Steps (Day 01)

1. Cài .NET 9 SDK
2. `dotnet new webapi -n QLNP.Api` → `packages/api/`
3. Add NuGet: FastEndpoints, EF Core SqlServer + Design + Tools
4. Cấu hình connection string trong appsettings.json
5. `dotnet ef dbcontext scaffold` → Entities/UserMaster.cs + Entities/DmDonvi.cs + AppDbContext.cs
6. Tạo 5 entity QLNP trong AppDbContext (OnModelCreating)
7. `dotnet ef migrations add InitialCreate` → exclude bảng cũ
8. `dotnet ef database update` → tạo 5 bảng mới trong DB
9. Middleware: CurrentUserMiddleware (placeholder)
10. `dotnet build` verify
