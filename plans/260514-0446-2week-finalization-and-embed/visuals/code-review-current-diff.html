Code Review
Tốt
Vertical slice thuần túy — mỗi operation có folder riêng (Create/List/Update/Delete), đúng pattern FastEndpoints.
Async FluentValidation DB check — MustAsync trong validator check uniqueness trực tiếp qua EF Core, không cần service layer.
Soft delete đúng cách — IsActive = false thay vì xóa cứng, check LeaveRequests FK trả 409 trước khi deactivate.
JWT RoleClaimType fix — thêm RoleClaimType = "Roles" vào Program.cs để Roles("quantri") hoạt động chính xác.
HTTP test file đầy đủ — có đủ happy path + error cases (DefaultDays=0, duplicate Code, 409 delete) trong QLNP.Api.http.
Cần xem xét
List endpoint: manual auth check — ListLeaveTypesEndpoint dùng HttpContext.User.Identity?.IsAuthenticated thay vì Policies()/Roles() trong Configure(). Nên dùng Authenticate(); trong Configure để nhất quán.
Create: DbUpdateException rộng — bắt DbUpdateException chung để xử lý duplicate Code. Nên check uniqueness bằng validator trước (đã có) → exception path chỉ là fallback, nhưng message "mã có thể đã tồn tại" hơi mơ hồ.
Delete: soft delete vs IsActive filter — List endpoint chỉ query Where(t => t.IsActive). Sau khi delete, record vẫn tồn tại trong DB nhưng không visible. Cần đảm bảo Update endpoint cũng check IsActive khi fetch.
Update: không check IsActive — FindAsync([id]) sẽ trả về record đã bị soft-delete. Nên thêm check leaveType.IsActive sau FindAsync.
Vấn đề
Thiếu unit/integration tests — không có test file nào cho LeaveTypes slice. Các edge cases (duplicate Code, FK constraint, JWT role guard) chưa được tự động hóa.
Delete: không check inactive record — nếu id trỏ tới record đã deactivate (IsActive=false), endpoint trả 204 No Content thay vì 404 hoặc 409.
Câu hỏi
Role name case-sensitive? — Roles("quantri") vs JWT claim value có đúng case không? Cần verify trong token payload.
List có cần phân trang? — hiện tại trả toàn bộ active records không giới hạn. Với dữ liệu nhỏ OK, nhưng cần document rõ giới hạn scope.
Migration đã được tạo? — LeaveType entity cần migration tương ứng. Files trong diff không thấy migration mới.
Description optional ở Update? — request có Description? nhưng entity set trực tiếp. Cần confirm overwrite với null là intentional.
